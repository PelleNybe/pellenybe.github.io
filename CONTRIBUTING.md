# Contributing to Corax CoLAB

First off, thank you for considering contributing to Corax CoLAB! It is people like you that make Corax CoLAB such a great tool.

## Where do I go from here?

If you have noticed a bug or have a feature request, make one! It is generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork Corax CoLAB and create a branch with a descriptive name.

## Implement your fix or feature

At this point, you are ready to make your changes. Feel free to ask for help; everyone is a beginner at first.

## Make a Pull Request

Finally, go to GitHub and make a Pull Request.

## Keeping your Pull Request updated

If a maintainer asks you to rebase your PR, they are saying that a lot of code has changed, and that you need to update your branch so it is easier to merge.
